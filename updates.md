> Этот файл создан для ввода в курс дела будущих AI-ассистентов. Содержит концепцию, текущее состояние, стек, структуру и дальнейшие шаги.

---

## 📌 О проекте

**Jas Volunteers** — централизованная веб-платформа для волонтёрского движения Казахстана (10 000+ участников, 20 команд).

**Проблема:** Организация работает через Google Forms + WhatsApp + разрозненные таблицы. Анонсы мероприятий вручную рассылаются по 20+ чатам. Учёт часов — хаос.

**Решение:** Единая платформа. Сайт — не замена WhatsApp для чатов. Он берёт на себя: доску объявлений, портфолио волонтёра, учёт посещаемости через QR, рейтинг команд.

## ✅ Что сделано

### Дизайн-система
- **Концепция:** «Тёплый минимализм» по `design-guide.md`
- **Цвет:** `#9B111E` (тёмно-красный), фон `#F7F8FA`
- **Шрифт заголовков:** Montserrat 800 Uppercase (как на логотипе)
- **Шрифт текста:** Inter 400–600
- **Логотип:** Казахский орнамент-сердце (`public/logo.png`), favicon обновлён
- **Иконки и Орнаменты:** Разработаны уникальные SVG-орнаменты (`Ornaments.jsx`) в стиле бренда (тёмно-красные казахские узоры в белых кругах) для отображения Рангов и Категорий мероприятий вместо банальных эмодзи.

### Маскот
- **MascotCat.jsx** — SVG-кошка в герой-костюме с плащом и полумаской
- 3 цвета шерсти: **белая / рыжая / чёрная** (переключатель с кнопками)
- **Гироскоп:** кошка наклоняется при повороте телефона
- **Shake:** встряхни телефон — кошка прыгает и говорит фразу
- **Hover (desktop):** кошка плавно следует за курсором
- **Клик/тап:** прыжок + речевой пузырь со случайной фразой на русском/казахском

### Страницы
- **HomePage:** Герой с маскотом, акцент на портфолио (не часы), ранги, карточки мероприятий из Supabase с mock-fallback
- **EventsPage:** Поиск + фильтры по категориям, карточки с прогресс-баром мест, кнопка «Записаться» (auth check)
- **TeamsPage:** Подиум топ-3 + таблица с рейтингом по часам (агрегация из profiles)
- **ProfilePage:** Реальные данные пользователя, ачивки, история мероприятий, ранг по часам, кнопки QR и PDF
- **LoginPage:** Email/пароль + Google OAuth, обработка ошибок, загрузка
- **RegisterPage:** Выбор роли (Волонтёр/Админ), выбор команды из Supabase, создание новой команды (через moderation), **toggle показа пароля + индикатор силы пароля**, жёлтая плашка с дружелюбным напоминанием писать настоящее имя

### Авторизация (AuthContext.jsx)
- `signUp` — регистрация с передачей роли и команды через `user_meta_data`
- `signIn` / `signInWithGoogle` / `signOut`
- `fetchProfile` — автоматически загружает профиль + данные команды после входа
- Auth state доступен всему приложению

### Layout (auth-aware header)
- Если **не авторизован** → кнопка «Войти»
- Если **авторизован** → аватар с инициалами + dropdown меню (профиль, панель управления по роли, выйти)

## 🗄️ База данных (supabase/schema.sql)

6 таблиц с RLS-политиками:

| Таблица | Назначение |
|---------|-----------|
| `profiles` | Профиль пользователя (роль, команда, часы, ранг) |
| `teams` | Команды (статус: pending/approved/rejected) |
| `events` | Мероприятия (дата, теги, max_participants) |
| `event_participants` | Записи волонтёров на мероприятия (status: registered/attended/absent) |
| `achievements` | Ачивки пользователей |
| `team_requests` | Заявки на создание новых команд (модерация Разработчиком) |

**Триггер:** при регистрации нового пользователя автоматически создаётся `profiles` запись с данными из `user_meta_data`.

> ⚠️ **Схему нужно применить:** Supabase → SQL Editor → скопировать `schema.sql` → Run

| Роль | Возможности |
|------|-------------|

## 🌟 Killer Features (запланировано)

1. **PDF-портфолио** — одна кнопка → красивый PDF документ. Строится из:
   - Благодарственные письма (БП)
   - Часы работы
   - Ранг (Новичок -> Легенда)
   - Роли за историю (например, "Был лидером команды осенью 2025")
   - **Уровень влияния (Impact Score):** График-диаграмма, показывающая в каких сферах волонтёр был наиболее активен (например, 60% — Экология, 30% — Животные, 10% — Донорство).
   - **Отзывы / Рекомендации:** Краткие теплые слова от лидеров команд или координаторов мероприятий.
   - **Галерея:** Лучшие фотографии с мероприятий, где участвовал волонтёр.

### Решение частых проблем (Troubleshooting)
- **Ошибка 429 (Too Many Requests) при регистрации Supabase Auth:** Это происходит из-за лимита Supabase (3 письма в час) на бесплатном тарифе для тестового почтового сервера. 
  - **Решение:** В админ-панели Supabase перейдите в `Authentication` -> `Providers` -> `Email` и отключите галочку **Confirm email** (Обязательное подтверждение почты). Это позволит регистрировать аккаунты без отправки реальных email-писем.

2. **Система Достижений (3 типа):**
   - *Классические:* Как сюжетка, доступны всем при выполнении условий (напр. "Посетил 5 эко-субботников").
   - *Особые:* Выдаются индивидуально админами за исключительный вклад (напр. "Финансовый вклад", "Спас человека", "Улучшил архитектуру").
   - *Событийные:* Уникальные ачивки привязанные к конкретному ивенту (напр. "Волонтёр месяца", "Чемпион по сборке", "Лучший координатор"). Выдаются админами.
3. **QR-код посещаемости** — у каждого волонтёра свой QR в профиле; Менеджер/Админ сканирует → автоматически засчитываются часы
4. **WhatsApp шаринг** — кнопка «Поделиться» при создании мероприятия генерирует отформатированное сообщение с deep-link на запись
5. **Leaderboard команд** — ежемесячный рейтинг по часам, создаёт здоровую конкуренцию. Включает уникальный лоры команд: (Ykiyo Crew, Sirius Team, Sonder Team, Shyraq Team, Gang Rise, Liberty Wave, Knight Team, Inspirit Team, Rinity Team, Scarlix Team и др.)
6. **Штрафная система** — предупреждение / ЧС / АЧС за неявку на записанное мероприятие
